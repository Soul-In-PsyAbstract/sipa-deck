Technical Audit Findings

What needs to change, in priority order

  • Critical
    No JSON-LD schema on service pages Google and AI engines cannot classify Medrisk as a medical service provider. MedicalOrganization + MedicalTest schema unlocks rich results and AI-confident citations.
  • Critical
    No llms.txt file ChatGPT, Claude, Perplexity, and Gemini scrape llms.txt to understand what a site does and who to recommend it to. Without it, Medrisk is invisible in AI-generated answers.
  • High
    Flat heading structure in Framer Current pages likely use visual hierarchy rather than semantic H1→H2→H3. Each demand-capture page needs one keyword-mapped H1 and structured H2 anchors for AI snippet extraction.
  • High
    No demand-capture pages for key job roles Searches like "pre-employment medical mining NSW" or "fitness for work construction Queensland" have intent but no Medrisk page to capture them.
  • Opportunity
    Resource hub template (ready for in-house content) A CMS collection with correct schema and article templates means your clinical team can publish directly — every post auto-inherits structured data and SEO defaults.

FRAMER IMPLEMENTATION APPROACH
──────────────────────────────────────────────────────────────

JSON-LD injection (per page):
  Site Settings → Custom Code → <head> (page-level overrides)
  Use Framer's "Page Variables" for dynamic values (service name,
  description) so CMS pages inherit schema automatically.

CMS Collection for Resource Hub:
  New Collection → "Articles"
  Fields: title, slug, excerpt, published_date, author, category
  Article template page with:
    - Auto H1 from CMS title field
    - Auto JSON-LD (Article schema) via head code template
    - Auto meta description from excerpt field
  → Editorial team publishes content; SEO defaults are enforced.

Heading structure (existing pages):
  Each service page: exactly one H1 (keyword-mapped, not logo text)
  H2 anchors: "What is X", "How it works", "Who needs it", "Get started"
  No decorative headings — every heading answers a real user question.

llms.txt placement:
  Framer → Project Settings → Custom Domain → Static Files
  Upload llms.txt as a static asset at root path /llms.txt
  Verify: https://medrisk.com.au/llms.txt

Alt text enforcement:
  Framer image components → Alt Text field required (non-empty)
  CMS image fields → alt as required CMS field
