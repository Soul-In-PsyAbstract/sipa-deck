Integration — Drop-In Replacement

One line change in Hire-a-Mind

SIPA OS is fully OpenAI-compatible. Your current interview analysis code changes only the base URL and key — nothing else.

POST api.sipa-os.org/v1/chat/completions 
// Before — OpenAI directly
const client = new OpenAI({ apiKey: "sk-..." });

// After — SIPA OS (same code, different endpoint)
const client = new OpenAI({
  apiKey: "sk-sipa-{tenant_key}",
  baseURL: "https://api.sipa-os.org/v1",
});

// Interview analysis — unchanged
const result = await client.chat.completions.create({
  model: "sipa-auto",
  messages: [
    { role: "system", content: "Extract structured hiring signals from this interview transcript. Return competencies, evidence, and confidence scores." },
    { role: "user", content: interviewTranscript },
  ],
  response_format: { type: "json_object" },
});

Each enterprise customer on Hire-a-Mind gets their own isolated sk-sipa-* key. Usage, billing, and rate limits are enforced per key — no shared state between tenants.
